# FY2025 功能使用年度总结 · 指标口径

- **财年**：2025-04-01～2026-03-31（与 `reference-tables.md`「公司财年」一致）。
- **时间粒度**：自然月；横轴为 12 个月。
- **数据（第 1 段 · query-business-metrics）**：`查询指标/FY2025_chart_data.json`（`meta.schema_version=2` 等结构由业务按需维护；**专用批量导出脚本已下线**，更新数据请按本文各模块口径自行查数后写入 JSON，或以 md/表格承载结论）。
- **图表（第 2 段 · business-charts）**：`图表/FY2025_功能使用年度总结.html`（`render_dashboard_html.py` + **`查询指标/FY2025_dashboard.manifest.json`**）。
- **技能内离线示例数据**：`business-charts/cache/example_dashboard_data.json`（与 FY2025 主数据文件无关，仅供 manifest 示例）。

## 1. 自定义报表（StarRocks `trace_log_dp`）

| 图表 | 口径 |
|------|------|
| 活跃度 | `event='PageView'`，`get_json_string(eventValues,'$.url')` LIKE `%/customDashboard/detail%`；工厂 `orgId` 在**当月**内 **`COUNT(DISTINCT dat) >= 8`**。 |
| PV | 同上条件下的 **PageView 条数**（`COUNT(1)`）。 |
| 户均 PV | **月 PV ÷ 当月「活跃度」工厂数**（活跃为 0 时记 0）。 |
| 留存率 | 上月**活跃**工厂集合 ∩ 本月**活跃**工厂集合，再 **÷ 上月活跃工厂数**（首月无留存）。 |

## 2. 多维进销存报表（同上埋点）

| 图表 | 口径 |
|------|------|
| 活跃度 | 同上 PageView，URL LIKE `%multiInventoryReport%`；**`COUNT(DISTINCT dat) >= 4`**。 |
| PV / 户均 PV / 留存 | 定义同第 1 节，仅 URL 与活跃阈值改为 4 天。 |

## 3. TV 版（`trace_log_dp`）

| 图表 | 口径 |
|------|------|
| 活跃度 | `event='tv-device-info'`；当月内该工厂 **事件总次数 `COUNT(*) >= 10`**。 |
| 平均活跃天数 | 当月内**至少 1 次** `tv-device-info` 的工厂，按厂统计 **`COUNT(DISTINCT dat)`**，再对厂取 **平均**。 |

## 4. 协同任务（ADB `dt_collaborative_task`，三实例 `org_id` 并集）

| 图表 | 口径 |
|------|------|
| 活跃度 | 当月 `created_at::date` 在窗内；每台子先取 `(org_id, 日)`，再在内存中**合并**各分片同一工厂的日期集合，**`distinct 自然日 >= 8`** 的工厂计入活跃。 |
| 留存 | 上月**活跃**∩本月**活跃**÷上月**活跃**（活跃定义同上）。 |
| 任务量 | 三分片 **`COUNT(1)` 相加**（当月创建的任务条数）。 |
| 平均任务/厂 | **月任务量 ÷ 当月活跃工厂数**。 |
| 关联任务比例 | **`SUM(associate_id IS NOT NULL)` ÷ 任务量**（分片分子分母相加后相除）。 |

软删等：与现有年度脚本一致，任务表当前 where 为 `TRUE`（若业务要求排除删除需另加条件）。

## 5. 委外管理（ADB `dt_outsource_order` / `dt_outsource_post`）

| 图表 | 口径 |
|------|------|
| 订单工厂数（高活跃） | 当月 `dt_outsource_order`：每台子取 `(org_id, created_at::date)`，**合并**各分片日期后 **`distinct 自然日 ≥ 8`** 的工厂（与协同「按天合并再判阈值」一致）。 |
| 「收货 ≥8 天」工厂 | 当月 `dt_outsource_post`：`post_type_name='收货'`、`COALESCE(deleted_at,0)=0`；每台子取 `(org_id, created_at::date)`，**合并**各分片日期后 **`distinct 自然日 >= 8`** 的工厂。 |
| 转化率 | **「收货 ≥8 天」工厂数 ÷ 高活跃订单工厂数**（分母为 0 时记 0）。 |
| 留存 | 上月**高活跃订单工厂**∩本月**高活跃订单工厂**÷上月**高活跃订单工厂**。 |
| 平均订单数/厂 | 当月订单**条数**（三分片 `COUNT(1)` 之和）**÷ 当月高活跃订单工厂数**。 |

### 关于「任务 8 天收货」的说明

「收货 ≥8 天」仍指 **「自然月内收货过账 distinct 日 ≥ 8」** 工厂数（跨分片合并日期）；若需改为 **「建单后 8 个自然日内首笔收货」** 等 SLA 口径，需补充 `order_id`/业务时间字段后再改 SQL。

## 脚本与联网

**取数**：按上文章节口径在 Archery 或使用通用脚本查数：

```bash
python3 .cursor/skills/query-business-metrics/scripts/run_archery_query.py \
  --instance <实例名> --table <表名> --sql "..." [--var KEY=VAL ...]
```

鉴权：`.cursor/skills/weekly-core-metrics/scripts/config.json`（`csrftoken` / `sessionid`）。

**出图**（在已维护好 `FY2025_chart_data.json` 的前提下）：

```bash
python3 .cursor/skills/business-charts/scripts/render_dashboard_html.py \
  --data 查询指标/FY2025_chart_data.json \
  --manifest 查询指标/FY2025_dashboard.manifest.json \
  --output 图表/FY2025_功能使用年度总结.html
```
