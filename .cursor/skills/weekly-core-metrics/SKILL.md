# Weekly Core Metrics

## 适用场景

当用户需要生成周报中的核心指标与洞察时，执行本技能。

## 固定指标范围

- 委外管理：
  - 覆盖客户数：一周内既创建过委外订单，又有收发记录的客户
  - 客户留存率：**真实周留存率** = 上周活跃客户中本周仍活跃的比例（**跨实例并集去重** 后计算：|上周活跃∩本周活跃| / |上周活跃|，见下节「多分片比率」）
  - 委外订单客户数：一周内有创建过委外订单的客户
  - 订单到收发货转换率：本周委外覆盖的客户数/本周委外订单客户数（有订单客户中产生收发记录的比例；原「仅下单」分母易导致比率超过 100%，故采用此口径）
- 协同任务：
  - 覆盖客户数：一周内有任意2天创建任务的客户
  - 客户留存率：**真实周留存率** = 上周活跃客户中本周仍活跃的比例（**跨实例并集去重** 后计算：|上周活跃∩本周活跃| / |上周活跃|，见下节「多分片比率」）
  - 创建协同任务客户数：一周内创建过任务的客户数
  - 活跃客户转换率：一周内有2天创建任务的客户数/一周内创建过任务的客户数
  - 从工单创建协同任务的比例：associate_id 不为空的任务数/一周内总的任务数（分片内计数后相加；若任务可跨分片重复，需按业务确认是否改为全局去重后再算）

## 多分片比率与跨实例去重（标准口径）

凡本技能中的 **xx 率 / 比例 / 转化率**（含 **客户留存率**、**订单到收发货转换率**、**活跃客户转换率**、**从工单创建协同任务的比例** 等），在数据分布于 **ADB 01/02/03** 时，**标准定义**为：

1. **先**在各分片按同一业务规则界定 **分子集合、分母集合**（或满足条件的实体 ID、记录数）；
2. **再**对 **`org_id`（或指标约定的实体键）在三个实例的结果做并集去重**，得到 **全局分子、全局分母**；
3. **xx 率 = 全局分子 / 全局分母**（分母为 0 时脚本内按约定处理）。

**留存率** 典型写法：全局 **上周活跃集合** \(L\) 与 **本周活跃集合** \(T\) 均为「各分片活跃 ID 的并集」，**留存率 = |L∩T| / |L|**。

**与当前 `generate_weekly_report.py` 的关系：** 脚本对多数指标采用 **分实例 SQL 出数，再对分子、分母分别相加**；在 **每个 `org_id` 只落在一个分片** 时，与上述 **并集去重** 口径 **数值一致**。若需严格按「先拉齐 ID 再算率」核对，可运行 **`.cursor/skills/weekly-core-metrics/scripts/compare_outsource_retention_deduped.py`**（委外留存对照）。若发现与周报差异，**以并集去重结果为准** 并反查是否存在跨片重复或口径漂移。

更一般的查数与比率约定见 **[query-business-metrics](../query-business-metrics/SKILL.md)** 中的 **「ADB 多分片：比率类指标」**。

## 周期规则

- 本周：本周一 ~ 本周日
- 近2周：上周一 ~ 本周日
- 本月：这个月第一天~最后一天

## 一句话触发

用户只需说：`生成本周核心模块周报`

## 执行步骤（技能内部自动完成）

1. 检查配置文件是否存在：`.cursor/skills/weekly-core-metrics/scripts/config.json`
2. 如果不存在，先从模板复制并填写：
   - `cp .cursor/skills/weekly-core-metrics/scripts/config.example.json .cursor/skills/weekly-core-metrics/scripts/config.json`
3. 重点检查 `auth.csrftoken` 和 `auth.sessionid` 是否为最新值（失效就更新）。
4. 运行脚本：
   - `python3 .cursor/skills/weekly-core-metrics/scripts/generate_weekly_report.py`
5. 脚本会自动输出到：
   - `周报/{xx月第x周周报}-杨士聪.md`（例如 `周报/03月第4周周报-杨士聪.md`）
6. 脚本会同时输出结构化数据：
   - `周报/latest_metrics.json`
7. 读取 `latest_metrics.json`，使用**当前对话的大模型能力**为每个指标生成：
   - `洞察结果`
   - `结论/改进建议`
8. 将 LLM 结果写回对应周报 markdown 文件，确保最终表格完整可直接使用。

## 输出格式

必须输出以下列：

`核心指标 | 指标口径 | 本周结果 | 上周结果 | 环比 | 洞察结果 | 结论/改进建议`

## 指标口径说明

- 委外“覆盖客户数”在当前脚本中定义为：周期内在 `dt_outsource_order` 有订单且在 `dt_outsource_post` 有收发记录的客户交集。
- 协同任务“覆盖客户数”定义为：周期内 `dt_collaborative_task` 按客户统计，至少 2 天有创建任务。
- 凡 **转化率、留存率、占比** 等 **率类** 指标，**全局口径**见上文 **「多分片比率与跨实例去重」**；与 **[query-business-metrics](../query-business-metrics/SKILL.md)** 中 **「ADB 多分片：比率类指标」** 保持一致。
- 如后续确认更精确字段（如收发货状态字段），可直接调整脚本 SQL。

## 注意事项

- 接口 token 会频繁失效；脚本报鉴权错误时，优先更新 `config.json` 的 token。
- 业务数据分布在 3 个实例，脚本会分实例查询并汇总；**凡「率」类指标**，口径以 **跨实例并集去重后再算** 为准（见上节），与 **query-business-metrics** 技能保持一致。
- 如需临时自定义输出路径，可加参数：`--output your_path.md`。
- 不依赖你自己的模型 API Key；分析步骤使用当前对话中的大模型完成。
